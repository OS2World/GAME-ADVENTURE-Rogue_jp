１．起動方法

　ゲームを開始するには、プロンプトに対して以下のいずれかのコマンドを入力する。

　ａ）ｒｏｇｕｅ

　　　通常のゲームを開始する。

　ｂ）ｒｏｇｕｅ　−ｓ

　　　スコアを見る。ゲームは開始しない。

　ｃ）ｒｏｇｕｅ　セーブファイル名

　　　指定したセーブファイルからゲームを再開する。

　ｄ）ｒｏｇｕｅ　−ｒ

　　　環境変数で設定したセーブファイルからゲームを再開する。


２．環境変数

　各種の環境変数を指定することにより、ゲームのパラメーターを変更できる。

　ａ）ＳＥＴ　ＨＯＭＥ＝ホームディレクトリー名

　　　ｃ）のオプションでゲームディレクトリーが設定されていない場合、ここで指定
　　　したホームディレクトリーにスコアファイルが置かれる。ホームディレクトリー
　　　も指定しない場合、スコアファイルはカレントディレクトリーに置かれる。

　ｂ）ＳＥＴ　ＵＳＥＲ＝プレイヤー名

　　　プレイヤー名を指定する。指定しない場合は『戦士』になる。目には見えないが
　　　スコアファイル上の記録はこのＵＳＥＲ変数で識別される。シングルユーザーの
　　　システムであるＭＳ−ＤＯＳではあまり気にしなくてよい。

　ｃ）ＳＥＴ　ＲＯＧＵＥＯＰＴＳ＝オプション，オプション，．．．

　　　ゲームの各種パラメーターを設定する。重複するパラメーターがあるときは、後
　　　のオプションで指定されたパラメーターが有効となる。なお、オプション文字列
　　　の中にコンマ『，』および等号『＝』を含めることはできない。

　　　オプション　　　　　　　　　　　意味　　　　　　　　　無指定時の値
　　　［ｎｏ］ｊｕｍｐ　　　　　　　　移動途中表示の有無　　ｎｏｊｕｍｐ
　　　［ｎｏ］ｐａｓｓｇｏ　　　　　　曲り角で止まらない　　ｐａｓｓｇｏ
　　　［ｎｏ］ｔｏｍｂｓｔｏｎｅ　　　終了時の墓標の表示　　ｔｏｍｂｓｔｏｎｅ
　　　［ｎｏ］ｃｏｌｏｒ　　　　　　　マップのカラー表示　　ｃｏｌｏｒ
　　　ｆｒｕｉｔ：＜フルーツ名＞　　　たまに拾うくだもの　　こけもも
　　　ｆｉｌｅ：＜セーブファイル名＞　セーブするファイル　　なし
　　　ｎａｍｅ：＜ニックネーム＞　　　空白を含んでもよい　　プレイヤー名
　　　ｄｉｒｅｃｔｏｒｙ：＜パス名＞　ゲームディレクトリー　なし
　　　ｍａｐ：＜表示色＞　　　　　　　表示色のマッピング　　ｃｂｍｙｇ
　　　ｔｙｐｅ：＜使用機種＞　　　　　使用機種名　　　　　　ｎｏｎｅ
　　　ｉｎｉｔ：＜開始列＞　　　　　　画面モードの開始列　　なし
　　　ｔｅｒｍ：＜終了列＞　　　　　　画面モードの終了列　　なし
　　　ｃｕｒｓｏｒ：＜オン／オフ列＞　カーソル消去文字列　　なし

　　　★ｊｕｍｐ
　　　　移動コマンドを実行したときに、移動が終了してコマンド待ちになるまで画面
　　　　の書き直しを行わない。ゲーム進行をスピードアップすることができる。

　　　★ｐａｓｓｇｏ
　　　　何かにぶつかるまで移動、あるいは何かを見つけるまで移動するコマンドを実
　　　　行したとき、通路の曲がり角では停止しないよう指定する。

　　　★ｔｏｍｂｓｔｏｎｅ
　　　　ｑコマンド以外によるゲーム終了時に、プレイヤーの墓標を表示する。

　　　★ｃｏｌｏｒ
　　　　ゲーム中のレベルマップを、『ｍａｐ』オプションで設定した表示色でカラー
　　　　表示する。『ｃｏｌｏｒ』あるいは『ｍａｐ』オプションのどちらかが設定さ
　　　　れていない場合にはモノクロームの表示となる。

　　　★ｆｒｕｉｔ
　　　　ゲーム中に食糧として拾うことがある果物の名前であり、好きな果物を指定す
　　　　ればよい。指定しなくともゲーム進行に影響はない。

　　　★ｆｉｌｅ
　　　　ゲームを途中でセーブする場合、このオプションで指定したファイルがセーブ
　　　　ファイルとして使われる。相対パス名を指定したときは、ゲーム開始時のカレ
　　　　ントディレクトリーからの相対パスとなる。このオプションを指定しない場合
　　　　には、セーブコマンドに対してファイル名の入力が要求される。このオプショ
　　　　ンはまた、セーブしたゲームを再開するときにも使われる。

　　　★ｎａｍｅ
　　　　ゲーム中のプレイヤー名となるニックネームを指定する。スコアファイルへの
　　　　登録にはこの名前が使われる。指定しなくてもゲーム進行に影響はない。

　　　★ｄｉｒｅｃｔｏｒｙ
　　　　ゲーム用のファイルが置かれるディレクトリーを指定する。ゲーム中はこのデ
　　　　ィレクトリーがカレントディレクトリーとなる。このオプションが設定されて
　　　　いる場合、スコアファイル、スクリーンダンプファイルはこのディレクトリー
　　　　に作成される。設定されていない場合にはスコアファイルはホームディレクト
　　　　リー、スクリーンダンプファイルはカレントディレクトリーに作成される。

　　　★ｍａｐ
　　　　『ｃｏｌｏｒ』オプションでカラー表示が設定されているとき、このオプショ
　　　　ンで指定する５文字の英字によって、壁と通路、床、品物、プレイヤーのそれ
　　　　ぞれに色を設定することができる。色には、ｗ（白）、ｒ（赤）、ｇ（緑）、
　　　　ｙ（黄色）、ｂ（青）、ｍ（マゼンタ）、ｃ（シアン）のいずれかを指定する
　　　　ことができる。大文字（ＷＲＧＹＢＭＣ）を指定したときは反転表示となる。
　　　　標準の設定は『ｃｂｍｙｇ』、すなわち壁と通路はシアン、床は青、怪物はマ
　　　　ゼンタ、品物は黄色、プレイヤーは緑となっている。

　　　★ｔｙｐｅ
　　　　使用機種を指定する。このオプションを指定することにより、各機種の機能に
　　　　応じて『ｉｎｉｔ』、『ｃｕｒｓｏｒ』オプションが適当に設定される。指定
　　　　できる機種名は以下のものである。

　　　　　　ｐｃ９８　　　　ＮＥＣ　ＰＣ−９８０１シリーズ
	　　ｐｃ１００　　　ＮＥＣ　ＰＣ−１００
　　　　　　ａｘ　　　　　　各社　　ＡＸ仕様マシン
　　　　　　ｆｍｒ　　　　　富士通　ＦＭＲシリーズ
　　　　　　ｂ１６　　　　　日立　　Ｂ１６シリーズ
　　　　　　ｉｆ８００　　　沖　　　ｉｆ−８００シリーズ
　　　　　　ｎｏｎｅ　　　　特定の機種を想定しない

　　　★ｉｎｉｔ
　　　　ゲーム開始時に設定される、カーソル形状の変更などの画面モード変更エスケ
　　　　ープシーケンスを指定する。指定しなくともゲーム進行に影響はない。実際に
　　　　指定する文字列は、エスケープシーケンスの各キャラクターのＡＳＣＩＩコー
　　　　ドを１６進２桁で表したものである。エスケープシーケンスの長さは最大２９
　　　　バイトである。

　　　　例えばＰＣ−９８０１シリーズに対して、ゲーム開始時に次のシーケンスを実
　　　　行したければ、

　　　　　　ESC ) 0 　　　　漢字を使用するモードを指示する
　　　　　　ESC [ > 3 l 　　２５行モードを指示する
　　　　　　ESC [ > 1 h 　　画面最下行をプログラムが使えるようにする

　　　　ｉｎｉｔを次のように定義すればよい。

　　　　　　init:1b29301b5b3e336c1b5b3e3168

　　　　なお、『ｔｙｐｅ』に『ｐｃ９８』を指定した場合、この文字列が自動的に定
　　　　義される。

　　　★ｔｅｒｍ
　　　　ゲーム終了時の画面モード変更エスケープシーケンスを指定する。指定の方法
　　　　は『ｉｎｉｔ』と同様である。

　　　　『ｔｙｐｅ』に『ｐｃ９８』を指定した場合、

　　　　　　ESC [ > 1 l 　　画面最下行をシステムが使用する

　　　　のシーケンスに対して

　　　　　　1b5b3e316c

　　　　という文字列が自動的に設定される。

　　　★ｃｕｒｓｏｒ
　　　　カーソルの表示をオン／オフするエスケープシーケンスがあれば指定する。こ
　　　　のオプションが指定されていれば、画面の書き直しを行うときにカーソルの表
　　　　示がオフにされる。このオプションも、指定しなくともゲーム進行に影響はな
　　　　い。『ｉｎｉｔ』と同様の文字列を『／』で区切ったものを指定する。エスケ
　　　　ープシーケンスの長さはそれぞれ最大９バイトである。

　　　　　　　カーソルオンの１６進表示／カーソルオフの１６進表示

　　　　『ｔｙｐｅ』に『ｐｃ９８』を指定した場合、

　　　　　　ESC [ > 5 l 　　カーソルを表示する
　　　　　　ESC [ > 5 h 　　カーソルを表示しない

　　　　のシーケンスに対して

　　　　　　1b5b3e356c/1b5b3e3568

　　　　という文字列が自動的に設定される。

　ｄ）ＳＥＴ　ＲＯＧＵＥＯＰＴ１〜ＲＯＧＵＥＯＰＴ９＝オプション，．．．

　　　環境変数ＲＯＧＵＥＯＰＴ１〜９には、ＲＯＧＵＥＯＰＴＳと同じオプションが
　　　指定できる。オプションが１行に収まらないとき、あるいはオプションを一時的
　　　に変更したいときに使うことができる。評価の順序は、ＲＯＧＵＥＯＰＴＳ、Ｒ
　　　ＯＧＵＥＯＰＴ１〜９の順である。


３．ＣＯＮＦＩＧ．ＳＹＳ

　ローグのオプションを設定するには、環境変数の領域をかなり消費する。このため、
　標準のＭＳ−ＤＯＳコンフィギュレーションでは環境変数領域が不足するおそれがあ
　る（標準の環境変数領域は１０パラグラフ、すなわち１６０バイトである）。ＭＳ−
　ＤＯＳ３．１以降を使っている場合には、ＣＯＮＦＩＧ．ＳＹＳファイルの設定によ
　り、この領域を広げるように指定することができる。このためにはＳＨＥＬＬコマン
　ドのＥスイッチを使う。例えば、２０パラグラフ（３２０バイト）の領域を取るため
　にはＣＯＮＦＩＧ．ＳＹＳに次の１行を追加する。

　　　SHELL=A:\COMMAND.COM A:\ /P /E:20		（MS-DOS3.1 の場合）
　　　SHELL=A:\COMMAND.COM A:\ /P /E:320	（MS-DOS3.3 の場合）

　なお、環境領域を広げない場合でも、ローグのコマンド『！（ＭＳ−ＤＯＳコマンド
　の実行）』を実行するのであればＳＨＥＬＬを設定しておく必要がある。このときは
　ＣＯＮＦＩＧ．ＳＹＳに次の１行を追加する。

　　　SHELL=A:\COMMAND.COM A:\ /P


４．標準的な設定

　次に、ＰＣ−９８０１シリーズをＭＳ−ＤＯＳ３．３で使用している場合に標準的と
　思われる、ゲーム用システムフロッピーの設定をあげておく。

　★ルートディレクトリーの内容

　　COMMAND .COM　　　　コマンドインタプリター
　　CONFIG  .SYS　　　　システム設定ファイル
　　AUTOEXEC.BAT　　　　起動時実行ファイル
　　GAMES       　　　　ゲームのＥＸＥファイルディレクトリー

　★ディレクトリーＧＡＭＥＳの内容

　　ROGUE   .EXE　　　　ローグのＥＸＥファイル
　　ROGUEDIR    　　　　ローグのゲームディレクトリー（最初は空である）

　★ＣＯＮＦＩＧ．ＳＹＳの内容

　　FILES=12
　　BUFFERS=20
　　SHELL=A:\COMMAND.COM A:\ /P /E:320

　★ＡＵＴＯＥＸＥＣ．ＢＡＴの内容

　　ECHO OFF
　　PATH A:\;A:\GAMES
　　SET USER=（名前）
　　SET ROGUEOPTS=type:pc98,directory:A:\GAMES\ROGUEDIR
　　SET ROGUEOPT1=file:A:\GAMES\ROGUEDIR\ROGUE.SAV
